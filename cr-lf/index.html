CR与LF(操作系统“下一行”的不同)

Posted on October 23, 2014

在最初的电报通讯时代，文本每读取到一行的末尾，便需要将纸张向下滚动一行（换行），并将打印头从最右边回到左边开头（回车）。下面这个图还是挺形象的：

因此ASCII字符中便有了LF：Line Feed(0x0A)，CR：Carriage Return(0x0D)。但现代计算机并不需要两个字符来表示下一行，因此在设计操作系统时大家就产生了分歧。为节省资源，有的设计者选择用一个字符表示。因为这个选择的不同，导致了不同OS的换行符的不同。

如下图，参考维基百科，Mac OS 2001年之前采用的是CR，之后用的是LF，Unix/Linux采用LF，而Windows采用的是CRLF。

CR: Carriage Return

在ASCII码中代表回车，符号：\r（return），十六进制码：0x0D（十进制码13，二进制码0000 1101）。一个字节来表示下一行。

LF: Line Feed

在ASCII码中代表换行符，符号：\n（newline），十六进制码：0x0A（十进制码10，二进制码0000 1010）。一个字节来表示下一行。

CRLF

Windows和Dos系统采用回车+换行两个字节来表示下一行。

互相转换
