Python+OpenCV教程番外篇1：代码性能优化

Posted on December 6, 2017

学习如何评估和优化代码性能。（本节还没更新完…………）

完成一项任务很重要，高效地完成更重要。图像处理是对图像的操作，数据量巨大，如果代码写的不好，性能差距将很大，所以这节我们来了解下如何测量和提升代码性能。

优化原则

  1. 数据元素少时用Python本身格式，数据元素多时用numpy：

所以numpy的运行速度并不一定比Python本身格式快，元素数量较少时，请用Python本身格式。

  1. 尽量避免使用循环，尤其嵌套循环，因为极其慢！！！

  2. 优先使用OpenCV/numpy中封装好的函数

  3. 尽量将数据向量化，变成numpy的数据格式

  4. 尽量避免数组的复制操作
