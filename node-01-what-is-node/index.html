【Mater Node】01. Node.js是什么？

Posted on May 24, 2020

Goal

  • 什么是Node？它能做什么？
  • Node跟浏览器有什么关系？

Main

OK，打开Node的官网：https://nodejs.org/，上面是这样介绍的：

Node.js是一个基于 Chrome V8 引擎的JavaScript运行环境。

Node.js is a JavaScript runtime built on Chrome’s V8 JavaScript engine.

嗯…啥意思呢？

浏览器

在2009年以前，JavaScript通常只用于编写浏览器里运行的网页交互代码，每个浏览器都包含一个JS引擎，它就是用来执行JS代码的，也就是把JS代码翻译成计算机能理解的机器代码。

参考维基百科，各个浏览器的JS引擎有所不同，如Chrome中的是V8，旧版edge里面的是Chakra等(微软已“叛变”🙃，新版edge基于Chromium，故也是V8引擎)。其中V8是综合性能最好的，另外大家也知道Chrome用的人也是最多的。

浏览器提供了一个JS的运行环境，通过编写JS代码，我们就可以操纵浏览器。比如写过网页的同学应该知道document和window对象，这就是浏览器提供的能力。

Node

后来，Ryan Dahl这位老哥就想，为什么不把JS引擎提取出来，然后在浏览器之外运行JavaScript呢？于是，他把Chrome中的V8引擎提取出来，然后集成在自己编写的C++程序中，最后把这个程序取名：Node

所以Node跟浏览器很像，只不过提供了不同的能力。比如用Node你可以访问文件系统、创建HTTP服务等等，这些能力在浏览器里是没有的。

现在大家应该能理解这句话了吧：

Node是一个基于 Chrome V8 引擎的JavaScript运行环境。

Node能做什么

我们知道浏览器里运行的网页程序称为客户端或前端，而数据存储和处理的重头是在服务端也就是后端。

所以你发现Node的好处了吗？原有的前端工程师不用学习一门新的编程语言就能编写后端的API，变身全栈工程师。而且前后端都使用JS的话，代码的整体一致性、管理性和可维护性会更高，因为都是同样的命名规范、同样的包工具、同样的代码实践方式等等。另外，Node有非常庞大的开源生态体系(NPM)，我们不用去编写一些基础的模块，而是更专注于自身的业务逻辑。

由于Node的设计架构，Node非常适合I/O密集型的中间层服务，具体我们下节再会～
